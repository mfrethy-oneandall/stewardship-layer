# Governance

## Maintainer

This project is maintained by **Mike Frethy** ([@mfrethy-oneandall](https://github.com/mfrethy-oneandall)).

## Decision-Making

The maintainer has final say on all project decisions including:
- Feature additions and removals
- API changes
- Release timing
- Contribution acceptance

## Contributions

Contributions are welcome via pull requests. See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.

The maintainer reviews all contributions and may:
- Accept as-is
- Request changes
- Decline with explanation

## No Committee Structure

This project uses a single-maintainer model. There is no voting, steering committee, or formal consensus process.

## Succession

If the maintainer becomes unavailable, the project may be archived or transferred. No formal succession plan exists at this time.
